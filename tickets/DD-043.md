# DD-043

## Title

Build `ddstartup` for systemd startup management.

## Method

TDD

## Acceptance

- `dashboard skills install ...ddstartup...` automatically provisions the startup unit through the skill install path with no extra user step
- `dashboard ddstartup.setup` and `dashboard ddstartup.enable` write or restore a systemd unit that runs `dashboard restart`
- non-root setup defaults to a user unit and root setup defaults to a system unit
- `dashboard ddstartup.status` reports the managed unit state through `systemctl`
- `dashboard ddstartup.logs` reads unit logs through `journalctl`
- `dashboard ddstartup.disable` disables DD auto-start without deleting the unit definition
- `dashboard ddstartup.remove` disables the managed unit and removes the installed unit file
- Docker tests, 100% coverage, docs, commit, and push are completed
