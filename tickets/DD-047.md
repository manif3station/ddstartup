# DD-047

## Title

Implement launchd and launchctl support beside systemd.

## Method

TDD

## Acceptance

- `ddstartup` supports systemd hosts through the existing `systemctl` and `journalctl` flow
- `ddstartup` supports macOS hosts through `launchctl` and launch agent or daemon plist management
- install-time auto-setup provisions DD startup on supported macOS hosts just as it does on supported systemd hosts
- `setup`, `enable`, `disable`, `status`, `logs`, and `remove` all work on both supported platform families
- Docker tests, 100% coverage, docs, commit, and push are completed
